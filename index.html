What is Cryptography ?


Few days back i got a problem like many of us have, problem was simple I forgot my employers website authentication password!
Then I alerted my search team inside my body to the brain for tracing it. Unfortunately my team disappointed me , so as no option i contacted my friends , colleagues. They redirected me saying no option left other than contacting "ADMIN DESK TEAM".
By EOD some how , after so many hiccups I was asked by "ADMIN DESK TEAM" to change the password and generate the new.

Later part of the day I was stuck by a question from my brain "Why can't that team just provide the 'same PWD' which I(myRAM) lost somewhere ?"

So after asking to Google mom , -- No the answer was not just the security concern.

Then what, so the chronology was like this,
In the earlier generation we just used to store the "PWD" as it is in the databases. Later we found that is not the safe practice if the attackers breach into DB then they can have the password.

So the developers sat together and told each other we will implement 'age old egyptian technique' of interpretting the data.(simply Cryptography)
At later point of time developers were not trusty enough amoung themselves and realised DBA-"those admin guys" can do some naughty stuff.

Now we can introduce our Hero into the play - 'Hashing technique of one-way compression functions' which means it is almost impossible to decrypt it. So the DBA guy can see the most hashed version of ur fancy password but not your original password 'Ex: Angel_Apu,Biryaniboy'.

This is one of the code level Hashing explanation of above functionality.
const saltRounds = 10; // based on number the algorithm applies on top of each time generated hash(encrypted msg)
const myPassword = 'handsomeboyRocky';//my real pwd
const someOtherPassword = 'normalRocky';//random guy attacking

bcrypt.genSalt(saltRounds, function(err, salt) {
bcrypt.hash(myPlaintextPassword, salt, function(err, hash) {
// Store hash in your password DB.
});
});

// now comapare the passwords by reverse technique //
bcrypt.compare(myPlaintextPassword, hash, function(err, result) {
// result == true (Now the system allows u to logon !)

bcrypt.compare(someOtherPlaintextPassword, hash, function(err, result) {
// result == false(u cant cheat me buddy !)
